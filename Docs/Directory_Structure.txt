Architecture notes:
- Keep the IDE shell and compiler pipeline decoupled.
- Treat `compiler/` as a language engine that can run from the terminal, tests, or the IDE.
- Add `semantic/` next, before expanding the VM, so name binding and diagnostics have a dedicated home.
- Move tests under `src/test/java` as the compiler surface grows.
